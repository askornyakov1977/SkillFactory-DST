# Проект 3. Предсказывание рейтинга отеля

## Описание проекта
Представьте, что вы работаете дата-сайентистом в компании Booking. Одна из проблем компании — это нечестные отели, которые накручивают себе рейтинг. Одним из способов обнаружения таких отелей является построение модели, которая предсказывает рейтинг отеля. Если предсказания модели сильно отличаются от фактического результата, то, возможно, отель ведёт себя нечестно, и его стоит проверить.


Условия:
- Решение оформляется только в Jupyter Notebook;
- Решение оформляется в соответствии с [ноутбуком-шаблоном](https://lms-cdn.skillfactory.ru/assets/courseware/v1/0a04828af83e98c62bf8782d920350d6/asset-v1:SkillFactory+DST-3.0+28FEB2021+type@asset+block/EDA_Project_3_model.ipynb)
- Участие в [соревновании на площадке Kaggle](https://www.kaggle.com/t/cb7841e866d743f6843848ce328b0034)
- Использование [датасета hotels.csv](https://drive.google.com/file/d/1Qj0iYEbD64eVAaaBylJeIi3qvMzxf2C_/view?usp=sharing).
- Выводы по каждому этапу оформляются в формате Markdown в отдельной ячейке;
- Качество кода (соблюдение стандартов оформления PEP-8, комментирование кода, README к проекту). Оформление проекта на GitHub, GitLab, Kaggle;
- Удаление строковых значений. Необходимо удалить из набора данных столбцы, данные в которых представлены не числами;
- Очистка от пропущенных значений (если таковые имеются). 
- Использование разных способов представления информации об одном и том же признаке;
- Использование дополнительных источников информации о расположении отелей, чтобы лучше охарактеризовать среду, в которой находится отель;
- Использование различных преобразований признаков;
- Разбор даты на несколько новых признаков;
- Использание отбора мультиколлинеарных признаков;
- Ивлечение полезных данных из строковых признаков: в текстах отзывов, адресах или тегах отеля может скрываться важная информация;
- Использание кодирования признаков как оцифровку некоторых строковых значений.
- Достижение максимальной точности модели, которая предсказывает рейтинг отеля. 
- Использование советов и подсказок менторов при выполнении проекта.
- Код должен быть читаемым и понятным: имена переменных и функций отражают их сущность, важно избегать многострочных конструкций и условий.
- Проект сдается на проверку, максимальный балл 21 (из них 0-3 балла — за каждое задание:
    * Качество кода (соблюдение стандартов оформления PEP-8, комментирование кода, README к проекту). Оформление проекта на GitHub, GitLab, Kaggle;
    * Очистка данных;
    * Исследование данных (качество визуализации, наличие идей, гипотез, комментариев);
    * Генерация признаков;
    * Отбор признаков;
    * Преобразование признаков;
    * Качество решения: результат метрики MAPE)
- Получение обратной связи от команды курса.

Что практикуем:
Использование площадки Kaggle для участия в соревнований по машинному обучению. Оформление проекта на GitHub в соответствии с требованиями. Работа с основными инструментами для работы дата-сайентиста: 
    IDE;
    GitHub;
    Jupyter Notebook;
    Kaggle.
